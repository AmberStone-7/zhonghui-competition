**关系总结：**

| 关系 | 说明 |
|------|------|
| contestants : works | **一对一** (1:1)，`works.contestant_id` 唯一约束 |
| works : votes | **一对多** (1:N)，一个作品可有多条投票记录 |
| works : scores | **一对多** (1:N)，一个作品最多 4 条评分 (每个板块一条) |
| admin_users : audit_logs | **一对多** (1:N)，管理员的操作日志 |

## 枚举类型

### admin_role

管理员角色枚举，用于 `admin_users.role` 字段。

| 值 | 说明 |
|------|------|
| `super_admin` | 超级管理员，拥有全部后台权限 |
| `scorer_a` | A 板块评分员 (品牌与活动规范，满分 4 分) |
| `scorer_b` | B 板块评分员 (视觉设计表现，满分 5 分) |
| `scorer_c` | C 板块评分员 (陈列专业度，满分 4 分) |
| `scorer_d` | D 板块评分员 (执行与细节，满分 2 分) |

### user_status

用户状态枚举，用于 `admin_users.status` 字段。

| 值 | 说明 |
|------|------|
| `active` | 正常可用 |
| `locked` | 已锁定 (连续登录失败 5 次或手动锁定) |

### work_status

作品状态枚举，用于 `works.status` 字段。

| 值 | 说明 |
|------|------|
| `pending` | 待审核 (刚提交) |
| `approved` | 已通过 (获得编号，参与投票和评分) |
| `rejected` | 已拒绝 (附带拒绝原因) |
| `deleted` | 已删除 (软删除) |

### scorer_role

评分员角色枚举，用于 `scores.scorer_role` 字段。

| 值 | 说明 |
|------|------|
| `scorer_a` | A 板块 - 品牌与活动规范 |
| `scorer_b` | B 板块 - 视觉设计表现 |
| `scorer_c` | C 板块 - 陈列专业度 |
| `scorer_d` | D 板块 - 执行与细节 |

### score_status

评分状态枚举，用于 `scores.status` 字段。

| 值 | 说明 |
|------|------|
| `unreviewed` | 未评分 |
| `reviewed` | 已评分 (可修改) |
| `locked` | 已锁定 (不可修改) |

## 表结构详情

### admin_users

管理员与评分员账户表。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | UUID | PK, default uuid4 | 主键 |
| `username` | VARCHAR(50) | UNIQUE, NOT NULL | 登录用户名 |
| `password_hash` | VARCHAR(255) | NOT NULL | bcrypt 哈希密码 |
| `role` | admin_role | NOT NULL | 角色 (super_admin / scorer_a~d) |
| `status` | user_status | NOT NULL, default 'active' | 账户状态 |
| `failed_login_count` | INTEGER | NOT NULL, default 0 | 连续登录失败次数 |
| `created_at` | TIMESTAMP | NOT NULL, default NOW() | 创建时间 |
| `updated_at` | TIMESTAMP | NOT NULL, default NOW() | 更新时间 (自动) |

**业务规则：**
- `failed_login_count >= 5` 时自动将 `status` 设为 `locked`
- 登录成功后 `failed_login_count` 归零
- 重置密码时若 `status = locked` 则自动解锁为 `active`

---

### contestants

参赛者信息表，与 works 表一对一关联。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | UUID | PK, default uuid4 | 主键 |
| `name` | VARCHAR(20) | NOT NULL | 参赛者姓名 |
| `address` | VARCHAR(255) | NOT NULL | 门店地址 |
| `tax_id` | VARCHAR(50) | UNIQUE, NOT NULL | 税号 (唯一标识门店) |
| `phone` | VARCHAR(20) | UNIQUE, NOT NULL | 手机号 |
| `created_at` | TIMESTAMP | NOT NULL, default NOW() | 报名时间 |

**业务规则：**
- `tax_id` 和 `phone` 均唯一，同一税号或手机号不可重复报名
- 与 `works` 表通过 `contestant_id` 一对一关联 (`uselist=False`)

---

### works

参赛作品表，与 contestants 一对一，与 votes/scores 一对多。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | UUID | PK, default uuid4 | 主键 |
| `contestant_id` | UUID | FK -> contestants.id, UNIQUE | 参赛者 ID (一对一) |
| `work_number` | VARCHAR(20) | UNIQUE, NULLABLE | 作品编号 (审核通过后分配，格式 ZH26-XXX) |
| `images` | TEXT[] (ARRAY) | NOT NULL | 图片 URL 数组 (1-3 张) |
| `status` | work_status | NOT NULL, default 'pending' | 作品状态 |
| `reject_reason` | TEXT | NULLABLE | 拒绝原因 (仅 rejected 状态) |
| `reviewed_at` | TIMESTAMP | NULLABLE | 审核时间 |
| `created_at` | TIMESTAMP | NOT NULL, default NOW() | 创建时间 |
| `updated_at` | TIMESTAMP | NOT NULL, default NOW() | 更新时间 (自动) |

**业务规则：**
- 报名创建时 `status = pending`，`work_number` 为 NULL
- 审核通过后分配编号 (自动递增 ZH26-001, ZH26-002, ...)，`status = approved`
- 只有 `approved` 状态的作品出现在公开展示和投票中
- 删除为软删除 (`status = deleted`)，不物理移除

---

### votes

投票记录表，记录每次公众投票。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | UUID | PK, default uuid4 | 主键 |
| `work_id` | UUID | FK -> works.id, NOT NULL | 被投票的作品 |
| `phone` | VARCHAR(20) | NOT NULL | 投票者手机号 |
| `ip_address` | VARCHAR(45) | NOT NULL | 投票者 IP 地址 (支持 IPv6) |
| `user_agent` | VARCHAR(500) | NOT NULL | 浏览器 UA 信息 |
| `created_at` | TIMESTAMP | NOT NULL, default NOW() | 投票时间 |

**唯一约束：**

| 约束名 | 字段 | 说明 |
|--------|------|------|
| `uq_vote_work_phone` | `(work_id, phone)` | 同一手机号对同一作品限投 1 次 |

**业务规则：**
- 数据库层: `(work_id, phone)` 联合唯一约束防重
- 应用层: 额外检查同一 IP 对同一作品的重复投票
- 导出时手机号脱敏显示 (前 3 位 + **** + 后 4 位)

---

### scores

评分记录表，每个评分员角色对每个作品最多一条评分。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | UUID | PK, default uuid4 | 主键 |
| `work_id` | UUID | FK -> works.id, NOT NULL | 被评分的作品 |
| `scorer_role` | scorer_role | NOT NULL | 评分员角色 (scorer_a~d) |
| `items` | JSONB | NOT NULL | 评分明细 (JSON 数组) |
| `subtotal` | FLOAT | NOT NULL | 板块小计得分 |
| `status` | score_status | NOT NULL, default 'unreviewed' | 评分状态 |
| `created_at` | TIMESTAMP | NOT NULL, default NOW() | 创建时间 |
| `updated_at` | TIMESTAMP | NOT NULL, default NOW() | 更新时间 (自动) |

**`items` 字段 JSONB 结构示例：**

```json
[
  { "item_name": "官方海报规范展示", "selected_score": 1.0 },
  { "item_name": "MP 产品陈列占比", "selected_score": 0.5 },
  { "item_name": "指定新品露出", "selected_score": 1.0 },
  { "item_name": "品牌识别度 - Logo 及视觉", "selected_score": 0.5 }
]
```

**业务规则：**
- 每个 `(work_id, scorer_role)` 组合最多一条记录
- `subtotal` 由后端根据 `items` 中各项 `selected_score` 自动求和
- `status = locked` 后不可修改 (返回 423)
- 总分计算: `score_a + score_b + score_c + score_d + popularity_score`，满分 20 分

### audit_logs

操作审计日志表，记录管理端所有关键操作。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | UUID | PK, default uuid4 | 主键 |
| `admin_user_id` | UUID | FK -> admin_users.id | 操作人 |
| `action` | VARCHAR(100) | NOT NULL | 操作名称 |
| `target_type` | VARCHAR(50) | NOT NULL | 操作对象类型 |
| `target_id` | VARCHAR(50) | NOT NULL | 操作对象 ID |
| `detail` | JSONB | NULLABLE | 操作详情 |
| `ip_address` | VARCHAR(45) | NOT NULL | 操作 IP |
| `created_at` | TIMESTAMP | NOT NULL, default NOW() | 操作时间 |

**已记录的 action 类型：**

| action | target_type | 触发场景 |
|--------|-------------|----------|
| `approve_work` | work | 审核通过作品 |
| `reject_work` | work | 审核拒绝作品 |
| `delete_work` | work | 删除作品 |
| `update_config` | config | 更新配置项 |
| `update_register_channel` | channel | 更新报名通道 |
| `update_vote_channel` | channel | 更新投票通道 |

---

### site_configs

键值对配置存储表，用于管理各种运行时配置。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | UUID | PK, default uuid4 | 主键 |
| `key` | VARCHAR(100) | UNIQUE, NOT NULL | 配置键名 |
| `value` | JSONB | NOT NULL | 配置值 (任意 JSON 结构) |
| `updated_at` | TIMESTAMP | NOT NULL, default NOW() | 更新时间 (自动) |

**预置/常用 key：**

| key | value 结构 | 说明 |
|-----|-----------|------|
| `register_channel` | `{"status": "open"/"closed", "start_time": "...", "end_time": "..."}` | 报名通道开关 |
| `vote_channel` | `{"status": "open"/"closed"/"not_started", "start_time": "...", "end_time": "..."}` | 投票通道开关 |
| `vote_closed_message` | `{"message": "投票已结束"}` | 投票关闭提示文案 |
| `vote_not_started_message` | `{"message": "投票尚未开始"}` | 投票未开始提示文案 |
| `popularity_score_config` | `{"tiers": [{"min": 0, "max": 100, "score": 1}, ...]}` | 人气分阶梯配置 |

**业务规则：**
- 采用 KV (Key-Value) 模式，`key` 唯一
- `value` 为 JSONB 类型，可存储任意 JSON 结构
- PUT 更新时若 key 不存在则自动创建 (upsert)
- 管理端更新配置时自动写入 `audit_logs`
