# 技术架构

## 项目概述

中汇文具 20 周年橱窗大赛系统是一个面向终端门店的全栈 Web 应用，用于举办"橱窗陈列大赛"活动。系统支持参赛者在线报名上传橱窗作品、公众投票、四位评委分板块评分、以及管理后台审核与数据导出等完整业务流程。

### 响应式设计策略

所有页面采用 `md:hidden` / `hidden md:block` 双布局模式：

| 端 | 断点 | 特点 |
|------|------|------|
| 手机端 | < 768px | bg-h5-video 全屏背景、H5 素材+Logo 堆叠 Hero、`min-h-screen` 确保背景满屏 |
| PC 端 | ≥ 768px | 纯图片 Banner、白色卡片布局、`max-w-[1104px]` 内容区 |

Layout 组件：
- **PC**: 隐藏品牌 → 中汇文具文字（非链接）+ 5 个图标导航
- **手机**: mobile-only 简化栏（Logo + "中汇文具 20 周年" 可点击回首页）+ 隐藏导航

### 数据授权流程

1. 用户访问任何需要授权的页面
2. `AuthGuard` 检查 `sessionStorage.data_authorized`
3. 未授权 → 跳转 `/auth` 授权页
4. 用户点击「确认授权」→ `data_authorized = "1"` → PC 跳 `/register`，手机跳 `/`

### Mock 降级机制

`src/api/mock.ts` 提供 12 个模拟作品数据：

- Showcase 和 Vote 页面先尝试调 API
- API 不可用时自动降级到 mock 数据
- 降级时页面顶部显示"当前为演示数据"提示

### 状态管理方案

| 方案 | 作用域 | 说明 |
|------|--------|------|
| `useState` | 组件内 | 表单、UI 状态、分页等局部状态 |
| `useAuth` Hook | 全局 (sessionStorage) | JWT token 与角色存取，登录/登出操作 |
| `useVoteStatus` Hook | 页面级 | 轮询投票通道状态 (open/closed/not_started) |
| `sessionStorage` | 浏览器会话 | 持久化 `admin_token`、`admin_role`、`data_authorized` |
| axios 拦截器 | 全局请求 | 自动注入 `Authorization: Bearer <token>` 请求头 |

### 评分体系

总分 = A 品牌分 + B 视觉分 + C 陈列分 + D 执行分 + 人气分 = **满分 20 分**

| 板块 | 评委 | 满分 |
|------|------|------|
| A - 品牌与活动规范 | scorer_a | 4 分 |
| B - 视觉设计表现 | scorer_b | 5 分 |
| C - 陈列专业度 | scorer_c | 4 分 |
| D - 执行与细节 | scorer_d | 2 分 |
| 人气分 (自动) | 系统 | 5 分 |

## 部署架构

### Docker 构建流程

```
1. 基础镜像: python:3.12-slim
2. 安装 Node.js 20.x
3. npm ci → npm run build -- --base /static/  (Vite 生产构建)
4. cp frontend/dist/* → /app/static/
5. pip install -r requirements.txt
6. CMD: bash startup.sh → uvicorn app.main:app
```

### 静态资源与 SPA 路由

- **开发环境**: Vite dev server `base: '/'`，Router `basename="/static"`，访问 `http://localhost:5173/static/`
- **生产环境**: Dockerfile 构建时传 `--base /static/`，HTML 引用 `/static/assets/...`，FastAPI 挂载 `/static` 目录
- SPA 回退: 所有非 API 路由返回 `index.html`，由 react-router-dom 接管

### 环境变量清单

| 变量 | 说明 |
|------|------|
| `DATABASE_URL` | PostgreSQL 连接串 |
| `SUPABASE_URL` | Supabase 项目 URL |
| `SUPABASE_KEY` | Supabase API Key |
| `SUPABASE_STORAGE_BUCKET` | 存储桶名称 |
| `JWT_SECRET_KEY` | JWT 签名密钥 |
| `JWT_ALGORITHM` | JWT 算法 (HS256) |
| `JWT_EXPIRE_HOURS` | Token 有效期 |

| `ALLOWED_ORIGINS` | CORS 允许的源 |
| `PORT` | 服务端口 (Railway 自动注入) |
