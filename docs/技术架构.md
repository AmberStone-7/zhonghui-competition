# 技术架构

## 项目概述

中汇文具 20 周年橱窗大赛系统是一个面向终端门店的全栈 Web 应用，用于举办"橱窗陈列大赛"活动。系统支持参赛者在线报名上传橱窗作品、公众投票、四位评委分板块评分、以及管理后台审核与数据导出等完整业务流程。

### 响应式设计

所有页面采用 `md:hidden` / `hidden md:block` 双布局：
- **手机 (< 768px)**: H5 背景图全屏、Hero 素材堆叠
- **PC (≥ 768px)**: 纯图片 Banner、白色卡片布局

### Mock 降级

`src/api/mock.ts` 提供模拟数据，API 不可用时自动降级，页面顶部提示"当前为演示数据"。

### 存储安全

`src/utils/storage.ts` 对所有 `sessionStorage` 操作提供 try-catch 保护，防止隐私模式/存储禁用导致白屏。

### 评分体系

总分 = A 品牌分 + B 视觉分 + C 陈列分 + D 执行分 + 人气分 = **满分 20 分**

| 板块 | 评委 | 满分 |
|------|------|------|
| A - 品牌与活动规范 | scorer_a | 4 分 |
| B - 视觉设计表现 | scorer_b | 5 分 |
| C - 陈列专业度 | scorer_c | 4 分 |
| D - 执行与细节 | scorer_d | 2 分 |
| 人气分 (自动) | 系统 | 5 分 |

## 部署架构

### 前端 (GitHub Pages)

- 构建: `npx vite build --base /zhonghui-competition/`
- 产物推送到 `gh-pages` 分支
- 公网: `https://amberstone-7.github.io/zhonghui-competition/`

### 后端 (Railway)

- Docker 构建: Node.js + Python
- 静态资源由 FastAPI serve
- 公网: `https://zhonghui-competition-production.up.railway.app`
