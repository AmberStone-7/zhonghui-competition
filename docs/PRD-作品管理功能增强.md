# PRD：作品管理功能增强 & 代金券标签

> 版本: v1.0
> 日期: 2026-06-25
> 状态: 待开发

---

## 1. 背景与目标

### 1.1 背景

中汇文具 20 周年橱窗大赛系统 V1 已完成基础功能：报名上传、作品审核、公众投票、四维度评分、数据导出。当前管理后台的作品管理功能较为基础——仅支持列表查看和删除操作，缺乏作品信息编辑、分数调整、以及与企业财务流程衔接的代金券管理能力。

### 1.2 目标

本次增强聚焦于两个核心需求：

1. **作品管理分区**：在管理后台左侧导航栏增加「已审核作品」「待审核作品」「待定作品」三个子分区，帮助管理员按审核状态快速定位作品。
2. **作品信息编辑 + 代金券标签**：主管理员可修改作品信息、调整评分、添加客编和备注，并创建代金券标签分配给作品，最终支持按标签筛选导出客户信息 Excel，供财务部门发放代金券。

### 1.3 权限范围

- 所有新增功能仅 **主管理员（super_admin）** 可操作。
- 评分员（scorer_a~d）不受影响，保持原有权限边界。

## 2. 功能需求

### 2.1 侧边栏导航改造

**现状：** 侧边栏为扁平结构，包含 报名审核 / 作品管理 / 内容管理 / 评分管理 / 数据导出 / 系统配置。

**改造后：**

```
管理后台
├── 报名审核          → /admin/review
├── 上传作品管理       → (分组标题，不可点击)
│   ├── 已审核作品     → /admin/works?status=approved
│   ├── 待审核作品     → /admin/works?status=pending
│   └── 待定作品       → /admin/works?status=hold
├── 内容管理          → /admin/content
├── 评分管理          → /admin/scores
├── 数据导出          → /admin/export
└── 系统配置          → /admin/config
```

**交互规则：**
- 「上传作品管理」为展开/折叠分组，展开后显示三个子项。
- 点击子项进入 `/admin/works` 页面并自动设置对应状态筛选。
- 高亮状态：当前选中子项高亮，父分组也保持高亮态。

### 2.2 作品管理页改造

**页面路径：** `/admin/works`

**顶部 Tab 栏：** 全部 / 已审核 / 待审核 / 待定（替代原有的下拉筛选）

**表格列（在原有基础上扩展）：**

| 列名 | 说明 |
|------|------|
| 编号 | work_number |
| 姓名 | contestant_name |
| 电话 | contestant_phone |
| 税号 | contestant_tax_id |
| 客编 | customer_number（新） |
| 状态 | status 徽标 |
| 代金券标签 | 标签名称 + 颜色圆点（新，无标签显示 "-"） |
| 提交时间 | created_at |
| 操作 | 编辑 / 删除 |

**编辑弹窗（点击「编辑」触发）：**

弹窗内分三个区域：

| 区域 | 内容 |
|------|------|
| **作品信息** | 可修改：作品编号、客编、备注信息。展示（不可改）：姓名、电话、税号、地址、作品图片 |
| **评分信息** | 展示四个维度当前分数（A/B/C/D + 人气分 + 总分），主管理员可直接修改各维度分数和人气分，总分自动重算 |
| **代金券标签** | 下拉选择已有标签（或创建新标签弹窗），可设置代金券金额和标签颜色，分配后展示当前标签信息 |

### 2.3 新增「待定」状态

**枚举值：** `hold`

**业务语义：** 审核员看过作品但暂时无法决定通过或拒绝（如图片不清晰、信息待补充等），先标记为"待定"。

**行为规则：**
- 待定作品 **不参与** 公众投票。
- 待定作品 **不参与** 评分。
- 待定作品 **不出现** 在公开展示页。
- 管理员可将待定作品重新审核为通过或拒绝。

**审核页改造：** 在通过 / 拒绝按钮旁新增「待定」按钮，点击后作品状态变为 `hold`。

### 2.4 代金券标签功能

#### 标签定义

| 属性 | 类型 | 说明 |
|------|------|------|
| name | 字符串(50) | 标签名称，如 "100元代金券" |
| amount | 数值 | 代金券金额（元） |
| color | 字符串(7) | 标签颜色，十六进制如 #FF6B6B |

#### 操作流程

1. 管理员在作品管理页点击「编辑」→ 切换到「代金券标签」面板
2. 可从下拉列表选择已有标签，或点击「新建标签」创建
3. 新建标签时填入名称、金额、选择颜色
4. 选择标签后点击「保存」，作品即关联该代金券标签
5. 可在编辑时移除或更换标签（设为空即移除）

#### 显示规则

- 标签 **不在前台（投票/展示/搜索）任何页面展示**，仅供管理后台内部使用。
- 管理后台作品列表中，用彩色圆点 + 标签名称展示。

### 2.5 代金券数据导出

**入口：** 管理后台 → 数据导出页 → 新增「代金券数据」导出卡片

**导出条件：** 筛选所有已关联代金券标签且状态为 `approved` 的作品。

**Excel 列结构：**

| 姓名 | 税号 | 客户编号 | 备注信息 | 代金券金额 |
|------|------|----------|----------|------------|
| 张三 | 91110... | KH001 | 橱窗陈列优秀 | 100 |

**支持格式：** CSV / Excel (xlsx)

## 3. 数据库变更

### 3.1 works 表新增字段

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| customer_number | VARCHAR(50) | NULLABLE | 客户编号（客编） |
| admin_remarks | TEXT | NULLABLE | 管理员备注信息 |
| voucher_tag_id | UUID | FK → voucher_tags.id, NULLABLE | 关联代金券标签 |

### 3.2 work_status 枚举新增值

| 值 | 说明 |
|------|------|
| `hold` | 待定（审核暂缓） |

### 3.3 新建 voucher_tags 表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | UUID | PK, default uuid4 | 主键 |
| name | VARCHAR(50) | NOT NULL | 标签名称 |
| amount | DECIMAL(10,2) | NOT NULL | 代金券金额 |
| color | VARCHAR(7) | NOT NULL, default '#6366F1' | 标签颜色 |
| created_at | TIMESTAMP | NOT NULL, default NOW() | 创建时间 |

## 4. API 变更

### 4.1 作品管理

| 方法 | 路径 | 说明 | 变更类型 |
|------|------|------|----------|
| GET | `/api/admin/works` | 支持 `hold` 状态筛选 | 修改 |
| PUT | `/api/admin/works/{work_id}` | 修改作品信息（编号/客编/备注/图片等） | **新增** |
| POST | `/api/admin/works/{work_id}/hold` | 设为待定状态 | **新增** |

### 4.2 分数调整

| 方法 | 路径 | 说明 | 变更类型 |
|------|------|------|----------|
| PUT | `/api/admin/scores/{work_id}/override` | 管理员手动覆盖各维度分数 | **新增** |

请求体示例：
```json
{
  "scores": {
    "scorer_a": 3.0,
    "scorer_b": 4.0,
    "scorer_c": 3.5,
    "scorer_d": 1.5,
    "popularity": 3.0
  }
}
```

### 4.3 代金券标签

| 方法 | 路径 | 说明 | 变更类型 |
|------|------|------|----------|
| GET | `/api/admin/voucher-tags` | 获取标签列表 | **新增** |
| POST | `/api/admin/voucher-tags` | 创建新标签 | **新增** |
| PUT | `/api/admin/voucher-tags/{tag_id}` | 修改标签 | **新增** |
| DELETE | `/api/admin/voucher-tags/{tag_id}` | 删除标签 | **新增** |
| PUT | `/api/admin/works/{work_id}/voucher-tag` | 给作品分配标签 | **新增** |

### 4.4 数据导出

| 方法 | 路径 | 说明 | 变更类型 |
|------|------|------|----------|
| GET | `/api/admin/export?type=voucher` | 导出代金券客户数据 | **新增** |
