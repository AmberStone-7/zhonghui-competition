## 1. 测试策略概述

### 1.1 三层测试结构

| 层级 | 类型 | 被测对象 | 工具 | 说明 |
|------|------|---------|------|------|
| **TDD (后端)** | API 接口测试 | FastAPI 路由 → 数据库 | pytest + httpx + AsyncClient | 每个接口独立测试，覆盖正常/异常/边界 |
| **SDD (前端)** | 组件 & 页面测试 | React 组件 & 页面结构 | Vitest + Testing Library | 验证 UI 渲染、交互行为、响应式断点 |
| **E2E** | 端到端集成测试 | 完整用户流程 | Playwright | 跨页面、跨角色完整流程验证 |

### 1.2 测试环境要求

- 后端: 独立测试数据库 (Supabase staging 或本地 SQLite/PostgreSQL)
- 前端: Vite dev server 或 生产构建后的静态资源
- 测试账号: 使用 seed 脚本预置的账号 (admin / scorer_a~d)
- 图片素材: 准备 1KB ~ 10MB 不同大小的 JPEG/PNG 测试图片

### 1.3 状态标记约定

| 标记 | 含义 |
|------|------|
| 🟢 | V1 已有测试覆盖，需补充 |
| 🔵 | V1 未覆盖，需新增 |
| 🟣 | PRD 新增功能，需完整测试 |
