## 三、核心操作流程

### 3.1 选手报名
选手访问首页 → 点击「报名上传」→ 填写姓名、地址、税号、电话 → 上传 1-3 张作品图片 → 提交报名。

### 3.2 审核作品（管理员）
1. 登录后台 `/admin/login`
2. 进入「报名审核」页面
3. 查看待审核作品，点击「通过」或「拒绝」
4. 审核通过后作品会分配编号（如 ZH26-001），并公开展示

### 3.3 评分流程（评分员）
1. 评分员登录 `/admin/login`
2. 自动跳转到评分任务列表
3. 逐个对已通过作品进行评分（选择对应分数项）
4. 评分后状态变为「已评分」
5. 管理员可锁定评分（锁定后评分员不可修改）

### 3.4 人气投票（公众）
访问 `/vote` → 选择作品 → 输入手机号 → 投票。
- 同一手机号对同一作品仅可投 1 次
- 同一 IP 对同一作品仅可投 1 次
- 投票通道可由管理员在后台开启/关闭

### 3.5 查看排名（管理员）
后台 → 评分排名：展示所有已评分作品的四维度得分 + 人气分 + 总分 + 排名。

### 3.6 数据导出（管理员）
后台 → 数据导出：
- **评分数据**: 作品编号、姓名、四维度分、人气分、总分、排名
- **报名数据**: 姓名、地址、税号、电话、作品编号、状态、时间
- **投票数据**: 作品编号、手机号（脱敏）、IP、时间
- **操作日志**: 操作人、操作类型、对象、详情、时间

## 四、部署与运维

### 部署平台
- **Railway** (Docker 自动部署): 推送代码到 GitHub 即自动触发构建部署
- **Supabase**: PostgreSQL 数据库 + 图片存储

### 环境变量（Railway Dashboard → Variables）
| 变量名 | 说明 |
|--------|------|
| `DATABASE_URL` | Supabase PostgreSQL 连接串 |
| `SUPABASE_URL` | Supabase 项目地址 |
| `SUPABASE_KEY` | Supabase anon key |
| `JWT_SECRET_KEY` | JWT 签名密钥（随机字符串） |
| `ALLOWED_ORIGINS` | CORS 允许的源（逗号分隔） |
| `FRONTEND_URL` | 前端地址 |

### 健康检查
`GET /api/health` → 返回 `{"status":"ok"}`

### 构建流程
Dockerfile 自动完成: 安装 Node.js → `npm ci` → `npm run build -- --base /static/` → 复制产物到 `/app/static/` → 安装 Python 依赖 → 启动 uvicorn

### 初始化说明
当前仓库中的 `startup.sh` 仅负责启动 `uvicorn`。数据库迁移（`alembic upgrade head`）和种子初始化（`python -m app.utils.seed`）需要在部署前或运维流程中单独执行。

### 故障排查

**页面 404**:
- 确认 Docker 构建成功
- 确认 `/app/static/index.html` 存在于容器中

**后台无法登录**:
- 确认数据库已完成迁移和种子初始化
- 连续输错密码 5 次后账号会被锁定，需由超级管理员重置密码解锁

**限流 429**:
- 投票/报名接口有频率限制（30次/分钟）
- 后台登录限制 5次/分钟（防暴力破解）
- 等待 60 秒后自动恢复

## 五、数据库备份

数据库托管在 Supabase，具有自动备份功能。如需手动备份：
1. 登录 Supabase Dashboard
2. 进入项目 → Database → Backups
3. 或使用 `pg_dump` 导出
